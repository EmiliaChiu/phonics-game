## 🎮 遊戲設計（v3.2 定案）

### 玩法總覽
2-4 隊分組競賽，輪流擲骰子在環狀棋盤上走格子，**誰先完整繞一圈回到起點就贏，遊戲立刻結束**（不是無限循環也不是線性終點）。

### 棋盤
- **12×12 環狀外圈，共 44 格**（`BOARD_N = 12`，`BOARD_LEN = 4×11 = 44`）
- **37 個核心注音符號保證全部上棋盤、不重複、不隨機抽樣**：21 個聲符（`CONSONANTS`）+ 16 個韻符（`FINALS_SINGLE`）＝ `CORE_SYMBOLS`。**結合韻不上棋盤**，只當進階/高階模式的抽卡內容
- 其餘非角落格：2 張命運卡格 + 1 個停車場
- 4 個角落固定：起點 / 停車場 / 命運卡 / 停車場
- 每次開新局 `buildBoard()` 重新洗牌位置，增加重玩變化性
- 格子顏色分類：🔴粉＝聲符、🔵藍＝韻符、🟡黃＝命運卡、⚪灰＝停車場、🟢綠＝起點

### 三種難度（開場選）
| 難度 | 玩法 |
|---|---|
| 初階版 | 走到格子念出上面的注音符號本身 |
| 進階版 | 走到「聲符」格 → 從該聲符所有合法的韻符+結合韻選項中隨機抽一張配對拼讀；走到「韻符」格 → 抽一張聲符卡配對拼讀 |
| 高階版 | 邏輯同進階版，額外抽一張聲調卡，要求用指定聲調念出來 |

- 拼讀配對**保證是真實存在的音節**，不會亂配：
  - `COMBO_SPEECH`：聲符+韻符 → 真實單一讀音字（例：`'ㄍㄜ':'歌'`），沿用自「正音23堂遊戲」
  - `COMBO3_SPEECH`：聲符+結合韻 → 真實單一讀音字（例：`'ㄍㄨㄛ':'鍋'`），這次新建，約150組
  - 兩份表分別衍生出 `CONSONANT_FINAL_OPTIONS`/`FINAL_CONSONANT_OPTIONS`（韻符方向）與 `CONSONANT_COMBINED_OPTIONS`（結合韻方向），落點時把兩個方向的選項合併成一個池子隨機抽
- **21 個聲符全部至少有 4 種合法配對**，不會有抽不到組合的情況（最少的 ㄖ 也有 4 種：3 組結合韻 + 1 組韻符）
- ㄝ / ㄦ / ㄧ 這三個韻符找不到合法配對聲符，會 fallback 成單獨念讀（這是國語本身的限制，找不到是正常，不是 bug）

### 聲調機制（高階版）
- **只從 `DRAWABLE_TONES` 抽（一聲/二聲/三聲/四聲，不含輕聲）**：輕聲是「第二音節相對弱化」的現象，單一個字獨立念本來就不會自然是輕聲，這是系統性問題所以整個拿掉，不是逐字修
- **抽到一聲時，畫面不顯示任何聲調卡或文字**（符合真實注音「一聲不標調號」的慣例），但計分仍當作有抽到聲調（+3分）
- **`TONE_EXCLUDE` 例外表**：處理「技術上有這個聲調的字，但太罕見/不合理」的個案，跟輕聲那種系統性問題不同。目前只有一筆：`'ㄎㄥ': [3]`（ㄎㄥ三聲kěng太罕見）。**這是可持續擴充的機制**——以後如果又測到某組合的某聲調很怪異，直接在這裡加一筆 `'組合鍵': [聲調id]` 即可（組合鍵＝聲符+韻符或聲符+結合韻兩個符號串起來）

### 命運卡（13 張效果卡，`FATE_CARDS`）
- 效果分三種宣告式欄位：`steps`（前進/後退格數，走動畫，跟骰子共用 `animateSteps()`）、`scoreDelta`（加減分，立即生效不用動畫）、`teleportStart`（瞬間退回起點，格數可能太多不用動畫）
- 抽到後**先顯示卡片文字，使用者點「繼續」才觸發走格動畫**（先看清楚效果、再看棋子真的走），走完才換下一隊
- 退回起點的命運卡（`teleportStart`）跟後退命運卡就算剛好走到 0 也**不算獲勝**，只有「往前走完整一圈」才算獲勝

### 停車場
- 落點會設 `team.skipNext = true`，這隊**下一輪會被自動跳過**（`nextTurn()` 用 while 迴圈檢查並清掉 flag），跳過那次後恢復正常
- 分數列上會顯示「😌休息中」提示大家這隊正在等待

### 勝負判定
- `animateSteps()` 裡偵測「往前走且從 `BOARD_LEN-1` 進位到 `0`」＝完成一圈，立刻呼叫 `winGame(team)` 結束遊戲、宣布獲勝隊伍
- `endGame(headline)` 是共用的結算畫面（顯示排名 + 自訂標題），`winGame()` 用它宣布獲勝者
- 老師的「🏁 結束遊戲看結果」按鈕保留當作**手動提早收尾**選項（例如下課時間到了但還沒人回到起點），呼叫 `finishGame()` → `endGame('老師提早結束遊戲...')`

## 🛠️ 技術架構

### 沿用「正音23堂遊戲」的資料與函式（手動複製，不引用外部檔）
- `PHONETIC_DATA`（58個注音符號的 word/emoji/speech）
- `COMBO_SPEECH`（聲符+韻符 真實配對表）
- `speak()` / `speakSymbol()`（TTS 發音規則）
- `vbopo()` / `bopoDisplay()`（直式注音顯示，ㄧ不旋轉）

### 本專案新增的關鍵資料/函式
- `CORE_SYMBOLS` / `FINALS_SINGLE`：37 個棋盤符號清單
- `COMBO3_SPEECH` / `CONSONANT_COMBINED_OPTIONS`：聲符+結合韻真實配對表（本專案新建，約150組）
- `buildBoard()`：37符號全上棋盤 + 命運卡 + 停車場，每局重新洗牌
- `boardCellRC(i)`：44格環狀座標轉換算式
- `resolveTile()`：依格子類型 + 難度分派邏輯的總開關
- `showSoloModal()` / `showPairModal()` / `showFateModal()` / `showRestModal()`：四種彈窗
- `DRAWABLE_TONES` / `TONE_EXCLUDE` / `drawToneFor()` / `toneDisplay()`：聲調抽取與例外機制
- `animateSteps(team, delta, onDone)`：走格動畫共用函式（骰子移動、命運卡效果都用它）
- `pulseTile()`：走格放大動畫
- `scrollCenterIntoView()` / `scrollTileIntoView()`：自動捲動邏輯
- `winGame()` / `endGame()` / `finishGame()`：結算相關
- `FATE_CARDS`：13張命運卡效果（`steps`/`scoreDelta`/`teleportStart` 宣告式欄位）

### 技術與資安規則
比照「正音23堂遊戲」同一套（詳見該專案規則檔的「🔒 資安規則」章節）：純前端 HTML+CSS+JS 單檔、無後端、無 CDN 依賴、不蒐集個資、localStorage 只存最近一局戰績（`zhuyin_monopoly_last_result`），不上傳任何資料到外部。
