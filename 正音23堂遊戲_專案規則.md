## 🔊 發音規則（極重要）

### 核心原則
**TTS（Web Speech API）對注音符號的處理：**
- 單一注音符號 → 老師端 TTS（Microsoft Hanhan/Yating）會念出該注音本身的音 ✅
- 兩個以上注音 → TTS 會拆開念，所以要改用代表合念音的中文字

### 三種發音策略

| 類別 | speech 欄位內容 | TTS 念出 | 範例 |
|---|---|---|---|
| **結合韻**（多注音組成一音節）| 一聲中文字 | 完整合念音 | ㄧㄢ → speech: `'煙'` → 念「yān」 |
| **單韻**（單一注音） | 注音符號本身 | 該韻的純音 | ㄢ → speech: `'ㄢ'` → 念「ㄢ」 |
| **聲符**（純子音） | 注音符號本身 | 該聲符的子音 | ㄅ → speech: `'ㄅ'` → 念「ㄅ」 |

## 🎨 視覺顯示規則

### 直式注音（vbopo）
- 所有注音符號採**直式顯示**（垂直堆疊）
- 結合韻例：ㄧㄢ 顯示為 `一` 在上、`ㄢ` 在下
- **ㄧ 維持橫線（一）形狀，不旋轉**
- 實作方式：`function vbopo(s)` 把字串拆成單字元，用 inline-flex column 堆疊
- CSS class：`.vbopo`、`.vbopo > span`

### 配色（兒童友善）
- 主色：粉紅 `#FFB6D9`、`#ff6b9d`（女孩友善）
- 副色：嫩黃 `#FFC857`、淡藍 `#A6D9F7`、嫩綠 `#7ED957`
- 背景：粉色到藍色到黃色的漸層

### 字體與按鈕
- 注音字大：認讀字卡 120px、按鈕 48px
- 按鈕圓潤、有圓角（border-radius: 16-20px）
- 答對動畫：彈跳 + 撒星星
- 答錯動畫：搖晃（不挫敗）

## 💾 進度系統

### 儲存方式：Plan B
- 主要：`localStorage`（key: `phonetic23_progress_v1`）
- 補充：頂部有「💾 匯出進度」按鈕（下載 JSON 檔）

- 補充：「🔄 重新開始」按鈕（清除所有進度，需確認）

### 進度結構
```json
{
  "lessons": {
    "1": {
      "flashcard": { "stars": 3, "score": 8, "completed": true },
      "memory": { "stars": 2, "score": 12, "completed": true },
      ...
    }
  },
  "wrong": {
    "ㄢ": 2,
    "ㄅ": 1
  }
}
```

### 星等規則（通過標準）
- 3 星：90% 以上正確（或無錯誤）
- 2 星：70% 以上正確（通過門檻）
- 1 星：低於 70%（建議回練習）
- 完成定義：所有 5 個遊戲都至少 2 星

## 🛠️ 技術架構

### 單一 HTML 檔結構
- 全部寫在一個檔案內（CSS + JavaScript inline）
- 無外部相依（不需要 CDN、不需要網路）
- 大小約 75 KB

### 主要 JavaScript 物件
```javascript
PHONETIC_DATA = {...}  // 58 個注音的資料
LESSONS = [...]        // 23 堂課
GAMES = [...]          // 5 個遊戲類型
progress = {...}       // localStorage 進度
```

### 主要函式
```javascript
goHome()                // 顯示首頁

goGame(lessonId, gameId)// 顯示某個遊戲
```

### 各遊戲實作函式
- `buildFlashcard()` - 認讀字卡
- `buildMemory()` - 記憶翻翻牌
- `buildListen()` - 聽音點點看
- `buildSequence()` - 順序記憶（取代舊版 `buildQuickReact` 快速反應／更早的 `buildWheel` 幸運轉盤）
- `buildConnect()` - 連連看
- `buildCombine()` - 拼讀組合（取代舊版 `buildMaze` 走迷宮）
- `buildTone()` - 聲調挑戰（第 22 堂）
- `buildFinale()` - 蚊子歌（第 23 堂）
- `buildQuiz()` - 闖關測驗
- `buildReview()` - 錯題複習（新增）

### 錯題記錄機制（新增）
- `progress.wrong = { 注音符號: 答錯次數 }`，不分堂次，全域累計
- `recordWrongAnswer(sym)` - 答錯時呼叫，累加次數
